最好的移动端测试工具[译]

发表于 2017-06-07   |   分类于 自动化测试   |  

引言

你是否正在寻找一种更先进的移动端测试方案？有很多种方式可以实现但是毕竟你的时间和精力有限无法一一了解。

即使你是移动APP测试的专家，在自动化测试上也一定存在着改进的余地。你需要知道采用什么方案以及使用什么样的工具。

移动端领域正在迅速的发展，移动APP包含的类型也是多种多样，例如从视频APP再到银行APP。这样以来测试移动APP会是一个相当复杂的过程。

对移动APP来说，平台、操作系统、测试场景以及网络环境都是多种多样的。

Android和iOS是最为流行的两个移动端操作系统。在这两个平台上有成千上万的APP需要被测试。

那么什么是移动端测试呢？

移动端测试是一个过程，这个过程是为移动设备（例如平板电脑，智能手机等）设计和开发一个可以进行一致性和可用性的应用程序。

移动端测试通常分为以下几个类型

  • 功能测试：根据需求规范检查应用的基本功能
  • 性能测试：测试客户端性能、服务器性能以及网络性能
  • 内存测试：与PC相比移动设备的内存是有限的，这项测试用来检测APP的内存优化情况
  • 中断测试：用来检测APP在运行期间受到其他情况导致运行中断的情况，例如来电、短信、低内存警告、低电量警告灯。
  • 安装测试：安装测试用来测试应用程序安装是否简单、平滑的，包括更新和卸载。
  • 可用性测试：它用来检测应用的效率、有效性以及满意度。

移动端自动化测试工具的类别

  1. 最佳的移动端测试工具
  2. 基于云服务的移动端测试工具和服务
  3. 移动APP闪退崩溃报告工具
  4. 移动端性能测试工具
  5. 移动设备上用来测试在线网站的模拟器
  6. 移动端A/B测试优化用具
  7. 移动端日志记录及测试管理工具
  8. 移动端测试服务及服务提供

最好的移动端测试工具

1. Appium（Android和iOS）

  • Appium是一款在iOS和Android测试原生APP、web应用以及hybird应用的开源的自动化测试工具
  • 对于使用Android或iOS SDk开发的APP也是一样适用的
  • Appium支持iOS上的Safari和Android上其他所有内置浏览器应用程序
  • 不需要修改任何APP的代码来进行测试，它适用于任何在真机或模拟器上运行的iOS或Android应用
  • 此工具仅适用于Android和iOS平台上运行的WebLOAD free edition程序
    下载地址:appium

2. WebLOAD – 移动端性能测试工具:


了解你的网站能够处理多少移动用户访问的唯一方法就是对您的网站做移动用户的压力测试。WebLOAD可以进行任何场景的测试，这样就能让您准确的了解Android和iOS用户体验。你可以在模拟器或者真机环境下进行测试（这样就能反映出真实的用户体验）

WebLOAD允许您通过真机设备，模拟器，JavaScript或直接使用WebLOAD的内置IDE来记录测试信息。 WebLOAD包含一个实时的Web仪表板，以便您可以跟踪任何设备上移动设备的负载测试。

免费版的WebLOAD支持50个虚拟用户，全部功能的测试。

下载地址:WebLOAD free edition

3. deviceConnect: 移动端测试管理工具

  • 麻雀虽小五脏俱全，deviceConnect由一个服务器、操作软件和一个架构设备组成
  • 十分安全，它可以部署您的企业防火墙后面
  • 基于Web的管理控制台允许工程师远程管理，跟踪和监控测试
  • 支持Android和iOS的智能设备远程连接和访问，支持和手动或自动化测试
  • 可靠的自动化工具推荐，Appium（Selenium），Tricentis，SOASTA，SeeTest，Calabash，HP的UFT，IBM的Rational以及大多数其他移动自动化工具

更多细节及下载地址:deviceConnect demo

4. Katalon Studio – 一个完整且免费的移动端及Web自动化测试工具

  • 全功能测试及执行管理
  • 为新手及高手准备的双脚本界面
  • 兼容Android和iOS的最新版本
  • 可以在模拟器、真机设备或云服务上执行脚本
  • 内置关键字的支持GUI，API和数据测试
  • 支持终端模式下的CI工作流程
  • 与JIRA，GIT和Kobiton移动云服务对接完善开箱即用

下载地址：Free Katalon Studio

5. Kobiton (iOS和Android的设备云服务)

  • Kobiton是一个移动设备云平台，可以访问真机设备，以便在本机，网络和混合Android/iOS应用程序上进行手动和自动测试
  • 建立在Appium开源框架之上
  • 不断向支持设备列表中最新的设备和操作系统版本
  • 跨设备测试无需脚本修改
  • 自动生成的运行日志，命令，截图和元数据可以更快地定位问题

更多信息：(Sign up to join the beta for Free)[https://kobiton.com/?utm_source=software_testing_help&utm_campaign=software_testing_help&utm_medium=pr&utm_content=ranking_5spot]

6. MonkeyTalk (Android和iOS):

  • MonkeyTalk可自动为Android和iOS进行功能性测试
  • 它是一个开源工具，由三个组件组成，包括IDE，脚本和代理
  • IDE使用录制和播放创建测试脚本，代理是是APP和测试仪库之间的连接纽带，脚本包括关键字，语法和Java执行引擎
  • 每一个小的“冒烟测试”都通过数据驱动，它适用于native、hybird和web应用

下载链接：MonkeyTalk

7. Calabash (Android和iOS):

  • Calabash是由Xamarin和Calabash维护的自动化测试框架，可与Selenium Web Driver进行比较
  • 它可以为native app进行自动化的功能测试
  • 它包含两个开源库，一个用于Android的开源库，另一个用于iOS native应用和hybird应用
  • 它提供专门用于触摸屏体验的API
  • 它适用于Ruby，Java，.NET，Flex和许多其他编程语言

下载链接：Calabash

8. iOS UI Automation(iOS):

  • iOS UI Automation是专为iOS应用开发的Apple开源测试自动化框架
  • 通过测试脚本对接口进行自动化的测试
  • JavaScript接口可以在设备UI上进行测试
  • 它与其他工具，方法和框架不兼容，因为它是专有工具
  • 它有助于减少软件产品开发所需的流程性工作和时间

更多信息：iOS UI Automation

9. UI Automator (Android):

  • UI Automator是一个开源框架，允许使用自动功能测试用例测试UI
  • 可以在一个或多个设备上运行
  • automatics.jar文件的/platforms/目录下，包含着UI Automator的接口
  • UI Automator框架使用JavaScript编写

更多信息：UI Aotomator

10. Robotium (Android)

  • Robotium是专为Android UI测试而开发的开源库
  • 它是一个Android测试自动化框架，为原生和混合应用程序提供支持
  • 它使用TestDroid和javascript脚本为Android应用进行自动化的黑盒测试
  • 通过测试脚本，它还允许编写功能，系统和用户验收测试场景
  • 基于测试脚本，它可以编写函数、系统和用户操作的测试场景

下载链接：Robotium

11. iOS Driver (iOS)

  • 该工具是一个开源工具，可以和Selenium完全整合，并使用Selenium / WebDriver API自动化测试iOS原生应用和混合应用程序
  • 这个工具在模拟器上的执行效率要比在真机设备上好，部分最新的设备可以在真机设备上运行，但是要比在模拟器上慢
  • 无需修改任何代码或者加载其他应用来测试设备上的app

下载链接：iOS Driver

12. KeepItFunctional (iOS):

  • KeepItFunctional（KIF）是用于功能测试的iOS集成测试框架，使用标准XCTest测试目标构建和执行测试用例
  • 它是一个开源框架，旨在测试移动应用UI，并允许iOS应用程序的自动化测试

下载链接：KIF

13. Selendroid (Selenium for Android):

  • Selendroid也是一个开源框架，他可以同时对多个设备和模拟器进行测试
  • 它由Selenium2客户端API编写，通过原生、混合及web应用的UI驱动测试
  • Selendroid的测试代码基于Selenium 2和WebDriver API
    下载链接：Selendroid

14. Ranorex (Android and iOS):

  • Ranorex是开源的UI自动化测试框架，用来测试Web和移动应用程序
  • Ranorex通常用来进行功能测试，以测试原生iOS应用程序，原生移动应用程序，iOS的移动网络测试，触摸测试等。
  • Ranorex支持验收测试，自动化测试，GUI测试，Web测试，移动测试，数据驱动测试，跨设备测试等。

下载链接：Ranorex
